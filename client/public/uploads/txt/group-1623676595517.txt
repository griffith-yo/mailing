Инициализация
git init

Добавление изменений файлов (файла) в коммит
git add .

Добавляем коммит для ветки, в которой находимся
git commit -m 'first commit'

Переключение на другую ветку
git checkout [yourbranchname]

Создание и мгновенное переключение на ветку
git checkout -b [yourbranchname]

Смерджим ветку main в ветку [yourbranchname].
git checkout [yourbranchname]
git merge main

РЕБЕЙС (Структура линейная, отсутствие лишних коммитов)
Переключись на [yourbranchname] и сделай rebase на main
git checkout [yourbranchname]
git rebase main


HEAD - это символическое имя текущего выбранного коммита
давай отделим HEAD от ветки bugFix и присвоим его последнему коммиту в этой же ветке.
git checkout [yourbranchname - ветки, в которой находишься в данный момент]

Переключение между коммитами
git checkout main^ - означает "первый родитель ветки main".
git checkout main^^ - означает прародитель (родитель родителя) main
 
Можно напрямую прикрепить ветку к коммиту при помощи опции -f. Например, команда:
git branch -f main HEAD~
Переместит (принудительно) ветку main на три родителя назад от HEAD.

Git Reset
git reset отменяет изменения, перенося ссылку на ветку назад, на более старый коммит.
Это своего рода "переписывание истории";
git reset перенесёт ветку назад, как будто некоторых коммитов вовсе и не было.
git reset HEAD~1 (на 1 шаг назад)
